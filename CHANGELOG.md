# 更新日志 (CHANGELOG)

## [1.8.4] - 2026-04-17
### 优化 (Optimization)
- **配置界面极致扁平化**：移除了冗余的“OCR 视觉模型 URL”与“文本模型 URL”独立输入框。现已统一合并为单一的 `API Base URL` 配置，极大幅度简化了 WebUI 的交互体验。
- **配置鲁棒性**：即便使用单一 URL，底层逻辑依然完美支持视觉中转与逻辑大脑的并行分流。

## [1.8.3] - 2026-04-17
### 特性 (Feature)
- **同步 Cline 代码修改**：整合了最新的多模态解析逻辑与配置定义。
- **配置架构优化**：优化了 `_conf_schema.json` 中的字段描述与校验规则。


## [1.8.2] - 2026-04-17
### 特性 (Feature)
- **支持引用消息的文本跨时空理解**：大幅强化对于 Reply 引用组件的底层解析架构。在保留原有提取图片的能力外，现已支持全面深度抓取引用的全部历史文字上下文。通过在 Prompt 中对 `【被引用的历史上下文】` 与 `【用户的当前指令】` 进行严格分区，使大模型完美具备了能够同时“看图看上下文”甚至直接回答“翻译这段话”的能力。

## [1.8.1] - 2026-04-16
### 修复
- **历史引用图片提取失效**：修复了在回复（Quote/Reply）包含图片的旧消息时，由于 OneBot/NapCat 的 `get_msg` 接口返回降级后的 CQ 码字符串（而非标准 JSON 数组）导致的正则解析忽略问题。现已引入“穿透取值 + CQ码提取”双引擎兜底解析，完美支持历史引用图片的学术渲染。

## [1.8.0] - 2026-04-16
### 重大变更
- **彻底取消闲聊功能**：应主人要求，插件现在彻底移除了直接文本回复的 `chat` 模式。无论用户是发送“你好”还是复杂的公式，所有内容都将被强制重定向至 PDF 渲染引擎进行学术化排版交付。

### 优化
- **鲁棒路由控制**：在逻辑大脑层面显式禁用了闲聊分支，并增加了“Chat to PDF”的自动转换层，确保所有 AI 响应流最终都能以文件形式到达用户终端。
- **系统提示词重构**：重写了 `system_prompt.txt`，将所有输出模式统一为单一的 PDF 交付模式。

## [1.7.3] - 2026-04-16
### 修复
- **Pydantic 类型校验崩溃**：`chat_messages` 列表中的元素在喂给 `event.plain_result()` 前，现在强制执行 `str()` 类型转换，彻底防止 LLM 返回了 `null` / 数字等非字符串类型导致的 Pydantic `ValidationError` 崩溃。
- **视觉 API 失败后的静默崩溃**：之前视觉 API 三次重试全部失败时，代码会带着空的 `image_description` 继续调用文本模型，并最终因 LLM 返回格式异常而 crash。现在已在重试耗尽时主动向用户输出明确的提示信息，并优雅地降级为纯文本问答，不再崩溃。

## [1.7.2] - 2026-04-16
### 修复
- **PDF 渲染超时问题锁定**：重构了 Playwright 对 MathJax 的生命周期监听逻辑。替换 jsdelivr 为访问稳定的大陆极速镜像 `elemecdn`，并且将原本脆弱的属性探测改为了原生的 Promise `pageReady` 钩子回调（`window.MATHJAX_DONE`）。这一组合彻底根治了偶发的 30 秒等待超时渲染崩溃问题。

## [1.7.1] - 2026-04-16
### 修复
- **区块级公式不渲染问题**：通过向 MathJax 运行时显式注入 `displayMath: [['$$','$$']]` 与 `inlineMath: [['$','$']]` 配置，修复了默认情况下 MathJax v3 无法识别通过 `$$` 包裹 LaTeX 进行分行渲染的 Bug。

## [1.7.0] - 2026-04-16
### 重构
- **PDF 渲染引擎现代化**：彻底移除对过时且维护停滞的 `wkhtmltopdf` (`pdfkit`) 的依赖，全面升级为基于 `Playwright` 的全异步浏览器渲染架构。显著提升了排版美观度、生成稳定性和渲染速度。
- **环境免配**：新架构原生完美支持苹果 M 系列 (Apple Silicon) 芯片，直接内置于 Python 环境，解决了此前复杂的自建依赖安装流程。

### 修复
- **公式渲染时序修正**：针对包含 LaTeX 数学公式的复杂解答，由于 Playwright 处理过快，引入了 `typesetPromise` 拦截阀。现在程序强制等待客户端的 MathJax 引擎渲染完所有前端 DOM 后才开始生成无头快照，彻底根治了导出 PDF 文件中出现原始 LaTeX 文本乱码的痛点。
- **弱链接剥离**：清理了代码内部失效的第三方 polyfill.io 脚本节点，提升页面挂载和导出响应速度。

## [1.6.10] - 2026-04-15
### 新增
- **兼容性大一统 (Greedy Match)**：实现了针对 OneBot/NapCat 协议的“多跳”图片匹配逻辑。不仅搜索标准 `url` 字段，还增加了对 `file`, `path` 等非标字段的贪婪兼容。
- **本地路径识别**：增加了对物理路径自动转 `file://` 协议的支持，大幅提升了在 NapCat 环境下的图片提取稳定性。

## [1.6.7] - 2026-04-15
### 修复
- **消息 ID 捕获深度适配**：针对 v4.23.1 中 `Reply` 组件字段名的隐蔽性，增加了对 `start_id` 和 `id` 字段的双重兼容探测。现在可以更稳定地从被回复的历史消息中精准“钓”出图片了。

## [1.6.6] - 2026-04-15

- **API 兼容性终极修正**：将错误的 `get_adapter` 调用更正为 v4.23.1 官方支持的 `get_platform_inst`，彻底解决了回复图片提取时的崩溃问题。

## [1.6.5] - 2026-04-15
### 修复
- **历史图片提取修复**：修复了在“回复”消息时，由于旧版 AstrBot `event` 对象缺少 `adapter` 属性导致的图片提取失败 Bug。现在通过 `self.context` 正确调度适配器。

## [1.6.1] - 2026-04-15
### 修复

- **稳定性增强**：清理了冗余的异常捕获逻辑。

## [1.6.0] - 2026-04-15
### 新增
- **视觉中转链路 (Vision Relay Chain)**：重构了多模态处理逻辑。现在视觉模型（如 Qwen-VL）仅负责纯粹的 OCR 提取，而精细的逻辑分析与 PDF 调度则交由逻辑更强的文本模型（如 DeepSeek）处理。这种“分工协作”模式显著降低了视觉模型的压力并提升了回答质量。

## [1.5.2] - 2026-04-15

## [1.5.1] - 2026-04-15
