# 更新日志 (CHANGELOG)

## [1.5.0] - 2026-04-15
### 新增
- **多模态智能分流引擎**：自动识别消息内容。纯文本调用 `llm_model`（如 DeepSeek），带图片调用 `llm_vision_model`（如 Qwen-VL）。
- **增强配置项**：在管理页面新增了视觉模型（Vision Model）的独立配置通道。
### 修复
- 解决了纯文本模型接收图片数据导致 API 报错 400 的物理限制。

## [1.4.2] - 2026-04-15
### 优化
- **极简说明书**：遵循简洁原则，将帮助信息重构为纯干货指令清单。

## [1.4.1] - 2026-04-15
### 新增
- **交互式帮助手册**：支持通过 `/ai help` 或发送空指令自动触发功能介绍。

## [1.4.0] - 2026-04-15
### 重构
- **大脑集成**：彻底移除对外部 8000 端口服务的依赖，将 LLM 路由逻辑直接内置于插件中。
- **可视化配置系统**：新增 `_conf_schema.json`，支持在 AstrBot WebUI 直接填写 API Key 和 URL。

## [1.3.3] - 2026-04-15
### 修复
- **API 标准化**：将 `await event.reply` 全部替换为官方规范的 `yield event.plain_result`。

## [1.3.2] - 2026-04-15
### 修复
- **属性路径兼容**：增加对旧版 AstrBot 消息对象 `.message` 属性路径的支持（适配 v4.23.1）。

## [1.3.1] - 2026-04-15
### 修复
- **版本探测与降级**：通过运行环境嗅探确认版本 API，锁定使用最稳健的 `@filter.command` 触发模式。

## [1.2.x] - 2026-04-15
- 初始重构与 Git 仓库初始化。
